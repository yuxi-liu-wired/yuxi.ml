# Ingress Answer (preview)

v0.9.1 (2025-06-29)

## 1  Overview

Chrome has long treated the omnibox as the single point of entry for navigation, but usage data shows that **56 %** of queries still require a context switch to a separate tab, site, or help article before the user acts. "Ingress Answer" is an experimental pane that lives directly under the right‑click (context‑menu) surface, providing a zero‑swipe path to *inline* documentation, code samples, and task‑oriented actions. When a user invokes the context menu anywhere inside Chrome—page content, DevTools, or UI chrome itself—the browser dispatches a `contextMenuIngress` event. If the event is not canceled by installed extensions, Chrome adds two new menu items:

1. **Ingress Answer** opens a lightweight pop‑over with top‑ranked answers for the current node, URL, or selection.
2. **Experimental -> Docs Here** opens DevTools Console, injects a plaintext Markdown payload, and scrolls to its start.

The feature is designed around three principles:

- **Focus stays in place**: No navigation; answers flow to the user.
- **Predictable performance**: A 200 kB Markdown budget guarantees sub‑100 ms paint.
- **Respect user control**: Both items respect existing policy hooks (`chrome.contextMenus`).

Developers can enable the experiment by loading Chrome with the flag
`--enable-features=IngressAnswerContextMenu` or by navigating to `chrome://flags` and toggling *Ingress Answer Context Menu* to **Enabled**. A silent Finch group (2 % Canary, 0.5 % Dev) began on 2025‑06‑10; wider release depends on crash telemetry and user opt‑out rates captured by UMA histogram `IngressAnswer.Enabled`.

## 2  Integration, API Surface, and Security Notes
### 2.1  Trigger Flow
When the right‑click occurs, Chrome assembles a *Resolver Context Object* containing:

```json
{
"location": "DOM|UI",
"url": "https://example.com/path",
"nodeName": "IMG",
"selection": "lorem ipsum",
"timestamp": 1656501123
}
````

This object is passed to the *Ingress Router*, a WebAssembly module compiled from Rust (`//components/ingress/router`). The router synchronously returns either an **Answer Card** payload (up to 6 KB) or a **Docs Pointer** payload (Markdown string, <= 200 KB). If the router returns *null*, Chrome hides both experimental menu items.

### 2.2  Extension Hooks

Extensions may override or augment the behavior via two new events:

* `chrome.ingress.onBeforeResolve` (blocking)
* `chrome.ingress.onAfterInject` (non‑blocking)

Both follow the Promise‑based pattern introduced in Manifest V3. To prevent a malicious extension from exfiltrating sensitive data, the context object is **redacted**: cookies, local storage keys, and credentials are stripped before dispatch.

### 2.3  Console Injection Mechanics

On **Experimental -> Docs Here**, the DevTools console opens in a new drawer session and runs:

```js
await chrome.ingress.injectMarkdown(payload);
```

`injectMarkdown` creates a `Blob` with MIME `text/markdown`, attaches it to a transient `<x-ingress-doc>` element, and logs it with `console.log('%o', element)`. The console viewer then treats the blob as a collapsed log group. Users can expand it, copy it, or right‑click -> *Save as…*. **Important**: Markdown is **not** rendered; the console deliberately shows raw text to preserve code fences and diagram source.

### 2.4  Privacy, Storage, and Cleanup

Markdown blobs are stored only in *renderer‑process* memory; closing DevTools or the tab triggers `IngressDocObserver::Dispose()` which revokes the object URL and zeroes its buffer. No disk write occurs unless the user explicitly saves. UMA histogram `IngressAnswer.BlobSize` tracks aggregate sizes (rounded to 4 KB buckets).

### 3.2  How to File Bugs

File issues under component **UI > Context Menus > Ingress Answer**. Please prepend the title with `[INGRESS]` and attach:

* `chrome://version` output
* `chrome://histograms` dump filtered by `IngressAnswer`
* Screenshots or GIF of the right‑click flow (redact private data)

### 3.3  FAQ

**Q:** *Why does the docs pane open in the console instead of a dedicated viewer?*
**A:** DevTools provides a trusted, sandboxed text viewer with built‑in save functionality and zero extra surface area.

**Q:** *Will the feature honor enterprise policy?*
**A:** Yes. Policy `IngressAnswerAllowed` (Boolean, default `true`) can disable both menu items.

**Q:** *Is offline content supported?*
**A:** Not in v0.9.1. The router requires network to fetch answers. Offline caching is on the 2026 roadmap.
